## Project Overview

Docusaurus v3.8.1 documentation + marketing site for KnowFlow, an enterprise knowledge base built on RAGFlow. TypeScript with React 19. All content is in Chinese (zh-Hans) with English i18n support.

## Development Commands

```bash
npm install          # Install dependencies (use npm, not yarn)
npm start            # Dev server with hot reload
npm run build        # Production build
npm run serve        # Serve production build locally
npm run clear        # Clear Docusaurus cache (useful when builds behave unexpectedly)
npm run typecheck    # TypeScript type checking via tsc
npm run deploy:aliyun  # Deploy to Aliyun server via deploy.sh
```

No linting or test framework is configured.

## Deployment

Two deployment targets:

1. **GitHub Pages** - Automated via `.github/workflows/deploy.yml` on push to `main`. Builds and deploys to GitHub Pages.
2. **Aliyun Server** - Manual via `npm run deploy:aliyun` (runs `deploy.sh`). Deploys to `139.224.59.152:/www/wwwroot/www.knowflowchat.cn` via rsync over SSH.

Production URL: `https://www.knowflowchat.cn` (configured in `static/CNAME`).

## Architecture

### Content Layer

- `docs/` - Documentation pages in Chinese, organized by topic folders (产品使用, 分块策略, API接口, 系统管理, 常见问题, 博客, 发布记录)
- `blog/` - Blog posts with reading time and RSS/Atom feed generation
- `sidebars.ts` - Uses `autogenerated` from `docs/` directory structure; folder ordering via `_category_.json` files

### Custom Pages (src/pages/)

Four marketing/product pages built as standalone React components with CSS Modules:

- **index.tsx** - Homepage: hero, capabilities, scenarios, testimonials
- **product.tsx** - Product details: features, tech specs, comparison table
- **about.tsx** - Company info: stats, values, team, partners
- **contact.tsx** - Lead generation form that submits to a WeChat Work (企业微信) webhook

Each page has a corresponding `.module.css` file for scoped styling.

### Utilities (src/utils/)

- **webhook.ts** - `sendToWeChatWork(formData)` sends contact form data to a WeChat Work webhook. Uses `no-cors` mode (cannot verify response server-side).

### Styling (src/css/)

- **custom.css** - Global theme overrides: Infima CSS variables, blue color palette (`#3b82f6` primary), navbar styling, dark mode adjustments

### Configuration

- **docusaurus.config.ts** - Site config including i18n, navbar items, footer, Prism themes, Baidu analytics integration
- **tsconfig.json** - Extends `@docusaurus/tsconfig`

## Key Patterns

- Sidebar structure is auto-generated from filesystem; use `_category_.json` in doc folders to control ordering and labels
- Page components follow a consistent section-based layout pattern (Hero -> Features -> CTA)
- CSS Modules (`.module.css`) for page-specific styles; global variables in `custom.css`
- Broken links set to `throw` (strict mode) - builds will fail on dead links
